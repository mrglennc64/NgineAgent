WHAT IT IS

One engine. Real telemetry. Four validated verticals.

The engine detects and corrects metadata that blocks CWR submission — split totals, missing ISWC/ISRC, missing writer IPI, society gaps, foreign-writer declarations, invalid role codes, and writer-name variants. NgineAgent makes every run observable:

  • Instrumented run-harness — wraps the engine, measures every step
  • Event stream (engine.events.v1) — append-only JSONL telemetry per run
  • Engine Run Report — execution timeline, step table, event log, performance metrics
  • N8N orchestration — importable workflows drive the engine via its HTTP API
engine.run.started  →  parse → detect → apply → detect (after) → render  →  engine.run.completed
